#  Sistema de ventanas X (X,x11,Xorg)

### ¿Que es xorg?

Es un sistema de ventanas muy usado en distros de linux, actua como intermediario entre el Hardware (pantalla, teclado, mouse) y las Apps graficas como (firefox, terminal, etc), no esta demas aclarar que X11, No es como GNOME, KDE, i3, sino que es la base el cual hace posible ver pantallas, este proyecto se considera un estandar de ventanas,  aunque se ha popularizado mas Wayland, que lo puede sustituir en un futuro 

"El predecesor del Sistema X Window fue un sistema de ventanas llamado W y fue un esfuerzo en conjunto entre IBM, DEC y MIT. Este software nació del Proyecto Athena en 1984. Cuando los desarrolladores empezaron a trabajar en un nuevo servidor de ventanas, eligieron la siguiente letra del alfabeto inglés: “X”. La evolución del Sistema X WIndow está actualmente controlada por el MIT X Consortium" (https://learning.lpi.org/es/learning-materials/102-500/106/106.1/106.1_01/)


Es sistema xorg si divide en dos partes importantes:
- Servidor de visualizacion: que proporcionan capacidades de visualización y realizan un seguimiento de la entrada del usuario, y los clientes, programas de aplicación que realizan tareas específicas. En un nivel más básico, el servidor actúa como un intermediario entre los programas de aplicación cliente y el hardware de visualización local
- Clientes: Los programas cliente realizan solicitudes (de información, procesos, etc.) que el servidor comunica a la pantalla del hardware


![alt text](/assets/2026-04-01/image.png)


**Servidor X de Visualizacion**
El servidor de visualización X es un programa que registra toda la información proveniente de dispositivos de entrada, como el teclado y el ratón, así como la de cualquier cliente que se esté ejecutando. A medida que el servidor de visualización recibe información de un cliente, actualiza la ventana correspondiente en la pantalla. El servidor de visualización puede ejecutarse en el mismo ordenador que el cliente o en una máquina completamente diferente


**Cliente X**
X permite ejecutar varios clientes simultáneamente. Por ejemplo, podrías estar editando un archivo de texto en una ventana, compilando el código fuente de un programa en otra, leyendo tu correo electrónico en una tercera y, al mismo tiempo, visualizando la carga promedio del sistema en una cuarta ventana.

Si bien los clientes X pueden mostrar sus resultados y recibir datos de un único servidor de visualización, cada uno puede estar ejecutándose en un equipo diferente de la red. Es importante tener en cuenta que los mismos programas pueden no verse ni funcionar igual en distintos servidores, ya que no existe una interfaz de usuario estándar, los usuarios pueden personalizar los clientes X de forma diferente en cada servidor y el hardware de visualización puede variar en cada uno

### Instalacion de Xorg 

**Nota:Esta instalacion es de arch linux, sin embargo, el proceso es el mismo**

```
pacman -S xorg
```


#### paquetes adicionales 

```
pacman -S xorg-xinit --> Inicio manual del servidor de Xorg

pacman -S xterm --> Terminal ligero, parte de X 

pacman -S xorg-xdm --> X display manager

pacman -S xorg-twm --> Window manager for Xorg


pacman -S xorg-xclock --> 

```


Podemos comprobar que se ha instalado de forma satisfactoria usando el comadno "xinit", el cual nos mostrara una terminal de forma grafica

Como bien sabemos, xorg, trabaja con ventanas y para saber que ventanas es quien, se usa la variable $DISPLAY, el cual contiene la numeracion de cada pantalla como :0, o si la pantalla esta remotamente renderizada, el formato es el siguiente: 


```
hostname:displaynumber.screennumber
```

donde:
- hostname: nombre del sistema que mostrara la aplicacion. Si no hay ninguno significa que es el entorno local
- displaymanager: Las pantallas que estan en uso o los servidores en uso (:0 primer servidor x, :1 segundo servidor, etc)
- screennumber: Por defecto es 0, (:0.0 pantalla 0 del display 0)


### Error comun al comunicarse con xorg con usuarios diferentes 

En caso de que recibamos un error como:

![alt text](/assets/2026-04-01/image-6.png)

Significa que no tenemos una cookie de sesion, en pocas palabras el archivo, .Xauthority, este nos permite ejecutar una sesion en el servidor X, actua como un sistema de autenticacion para verificar que el usuario esta autorizado para acceder al display manager 

Para ello generamos podemos hacer una de las siguientes opciones 

- Compartir la Cookie a otros usuarios (uso de cp)
- Generar la cookie compartida por defecto (/etc/skel)
- Crear una cookie (xauth)
- Agregar al usuario sin Cookie (xhost)

**Compartir la Cookie a otros usuarios (uso de cp)**
Para compartir la cookie con el usuario simplemente copiaremos el archivo .Xauthority al directorio, despues cambiaremos el usuario y grupo del archivo, como en la siguiente captura


![alt text](/assets/2026-04-01/image-7.png)

Se intento ejecutar el software xclock, que muestra la hora en formato analogico, sin embargo como no hay cookie para comunicarnos al servidor X, compartirmos la del usuario leonardo al home del usuario userOne para poder acceder al servidor, para finalmente volver a ejecutar el programa y conseguirlo, con esta cookie podremos usar llamadas al servidor, con comandos como (xclock, xload, xhost, xauth,etc)


**Generar la cookie compartida por defecto (/etc/skel)**
Para ello simplemente con el usuario root, copiaremos la cookie del usuario que tiene la cookie, al directorio /etc/skel, de esta manera, al crear un usuario, la cookie aparecera por defecto en el directorio home.

Ejecutamos xclock para comprobar la comunicacion al servidor xorg

![alt text](/assets/2026-04-01/image-8.png)

**Crear una cookie (xauth)**


```
# Generar la cookie 
xauth generate $DISPLAY . trusted

# listarla 
xauth list $DISPLAY

# exportarla 
xauth extract - $DISPLAY > /tmp/x11cookie

```

Iniciamos sesion con el usuario al que obtendra la clave y ejecutaremos lo siguiente 

```
$ su - otro_usuario
$ xauth merge /tmp/x11cookie

```


**Agregar al usuario sin Cookie (xhost)**
Este metodo es mas inseguro ya que a diferencia de xauth, este metodo no necesita de una clave, cookie o contraseña que verifique que el usuario es legitimo, simplemente se usa agrega a un usuario a una lista, al que simplemnte el sevidor confira, para hacerlo usamos:


```
# Listar usuarios
$ xhost 


# Agregar un usuario
$ xhost +SI:localuser:USUARIO


# Eliminar un usuario
$ xhost -SI:localuser:USUARIO
```




### Comandos 

xinit - inicia una sesión básica de X11 manualmente  
startx - inicia el entorno gráfico utilizando .xinitrc  
xdpyinfo - muestra información detallada del servidor X (resoluciones, extensiones, etc)  
xwininfo - muestra información de una ventana específica (tamaño, posición, ID)  
xlsclients - lista las aplicaciones (clientes X) conectadas al servidor  

xrandr - gestiona resolución, monitores y orientación de pantalla  
xinput - lista y gestiona dispositivos de entrada (teclado, mouse, touchpad)  
setxkbmap - cambia la distribución del teclado  

xprop - muestra propiedades internas de una ventana  
wmctrl - permite listar y manipular ventanas (mover, maximizar, etc)  

xhost - gestiona el control de acceso al servidor X (lista o permite usuarios)  
xauth - gestiona cookies de autenticación (.Xauthority)  

xeyes - aplicación gráfica de prueba para verificar X11  
xclock - reloj gráfico simple para pruebas  
xlogo - muestra el logo de X (prueba básica del servidor)  

glxinfo - muestra información de OpenGL (aceleración gráfica)
